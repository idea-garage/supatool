# Supatool design

See the [README](../README.md) and [tool-overview.md](./tool-overview.md) for overview and commands.

## 1. Directory layout (example)

```
supatool/
  bin/            # CLI entry
  src/            # Core logic
    parser/       # YAML/JSON/TS parsers
    generator/    # SQL/CRUD/types/RLS/doc generation
    templates/    # Code templates
  docs/           # Documentation
  tests/         # Unit tests
```

## 2. Input spec

- Data model is described in YAML/JSON or TypeScript types.
- Example: model.yaml

```yaml
entities:
  - name: user_profiles
    description: User profile
    fields:
      - name: id
        type: uuid
        primary: true
      - name: name
        type: text
        notNull: true
    relations:
      - type: hasMany
        target: participants
```

## 3. Core capabilities

### 3.1 Parse

- YAML/JSON/TS → JS object

### 3.2 Generate

- SQL: CREATE TABLE, FKs, RLS
- TypeScript types: DB type → TS type mapping
- CRUD: typed supabase-js helpers
- Docs/API spec: Markdown, OpenAPI, etc.

### 3.3 CLI

- Examples: `supatool gen:sql model.yaml`, `gen:crud`, `gen:types`, `gen:rls`, `gen:docs`

## 4. Extensibility

- Plugin-style generators
- Template override

## 5. Testing

- Unit tests (e.g. Jest)
- Snapshot tests for input → output

## 6. Error handling

- Input validation
- Detailed errors on generation failure

## 7. Future ideas

- ER diagram generation
- UI generation
- DB migration generation

## 8. Seed feature (v0.3.5+)

- Fetches data from selected tables (schema-aware) and saves as AI-friendly seed JSON.
- Tables are listed in a YAML file (e.g. tables.yaml).

```sh
supatool seed --tables tables.yaml
```

- Output: `supabase/seeds/<timestamp>_supatool/{table}_seed.json`
- Format:

```json
{
  "table": "public.users",
  "fetched_at": "2024-06-07T12:34:56Z",
  "fetched_by": "supatool v0.3.5",
  "note": "This data is a snapshot of the remote DB at the above time. For AI coding reference. You can update it by running the update command again.",
  "rows": [ { ... }, ... ]
}
```

This is a snapshot of the remote DB; you can refresh it by re-running the command.

### tables.yaml example

```yaml
# Default schema is public
tables:
  - users
  - posts

# With schema
tables:
  - public.users
  - app_data.logs
```

#### llms.txt (AI seed index)

- `llms.txt` is written under `supabase/seeds/` (overwritten each run).
- It is not written inside subfolders.
- After seed export, a timestamped folder is created and `llms.txt` is written there, indexing all `*_seed.json` files in that folder.

Example:

```
# AI seed data index (generated by supatool)
# fetched_at: 2024-07-05T11:16:00Z
# folder: 20240705_1116_supatool
public.users: users_seed.json # User accounts (2 rows)
public.orders: orders_seed.json (5 rows)
```
